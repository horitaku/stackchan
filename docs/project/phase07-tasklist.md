# フェーズ 7 タスクリスト（WebUI と可観測性）

## 1. このドキュメントの目的

フェーズ 7（WebUI と可観測性）を実行しやすくするために、作業を具体タスクへ分解して管理します。
本ドキュメントは「日次で更新する実行用リスト」です。

## 2. 運用ルール

- ステータスは `Planned`、`In Progress`、`Blocked`、`Done` を使用します。
- 作業開始時に `開始日` を記録し、完了時に `完了日` を記録します。
- `Blocked` になった場合は、必ず `ブロック理由` と `解除条件` を記載します。
- 1 タスクは原則 0.5 日から 2 日で終わる粒度に保ちます。

## 3. フェーズ 7 完了条件

次のすべてを満たしたらフェーズ 7 完了とします。

- 接続状態、再生状態、会話パイプライン遅延、アバター同期状態を取得できるサーバー API が提供されている。
- 設定変更 API（音量、表情プリセット、リップシンク係数、モーション有効化）が実装されている。
- WebUI の最小ダッシュボードで接続状態、再生状態、主要レイテンシが可視化されている。
- WebUI から疎通テスト（STT/LLM/TTS の最小確認）を実行し、結果を表示できる。
- 閾値ベースのアラート表示（遅延、失敗件数、接続状態）が WebUI で確認できる。
- `session_id` と `request_id` をキーに、診断情報を追跡できる。
- API と WebUI の主要フローに最低限の自動テストが追加されている。
- `server/README.md` と `docs/` に運用手順と確認手順が記録されている。
- フェーズ 8（統合と運用）へ引き継ぐ未解決事項と改善優先度が記録されている。

## 6. 前提と設計メモ

### フェーズ 6 からの引き継ぎ前提

| 項目 | 状態 | 備考 |
| --- | --- | --- |
| `tts.end` の PCM 再生 | 実装済み | firmware で再生確認済み |
| リップシンク（振幅ベース） | 実装済み（最小） | 係数調整はフェーズ 7 で UI 化 |
| `avatar.expression` / `motion.play` 受信 | 実装済み（最小） | 表示/制御の可視化は未整備 |
| request_id 伝搬 | 実装済み | 可視化 API で露出が必要 |
| heartbeat 継続動作 | 実機確認済み | 接続可視化へ反映が必要 |

### フェーズ 7 の初期設計方針

- WebUI は「設定変更」と「診断確認」を明確に分離する。
- 可視化 API は読み取り専用を優先し、設定 API は書き込み専用で分離する。
- 指標はまず最新値中心で表示し、履歴表示はフェーズ 8 で拡張する。
- アラートは閾値ベースの最小実装から開始し、運用値は後続で調整する。

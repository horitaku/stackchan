# 機密情報の運用ルール

## 目的

この文書は、Stackchan 再構築プロジェクトで扱う機密情報を安全に運用するための最小ルールを定義します。

## 対象

- API キー
- Wi-Fi 認証情報
- WebSocket トークン
- DB パスワード
- 証明書、秘密鍵

## 共通禁止事項

- 実値を Git にコミットしない
- チャット、Issue、PR に平文で貼り付けない
- ログに機密値を出力しない
- サンプルファイルに実値を書かない

## ローカル開発

- server は `.env.example` を `.env` にコピーして利用する
- firmware は `firmware/platformio.ini.example` と `firmware/include/secrets.example.h` からローカル専用ファイルを作成する
- ローカル専用ファイルは `.gitignore` で除外する

## CI 環境

- CI の Secret ストアから環境変数注入する
- ワークフロー内に平文キーを直接記述しない
- 必要最小権限のキーを利用する

## 本番環境

- Secret Manager または同等機能から起動時注入する
- 定期ローテーションを実施する
- 失効手順を運用手順として保持する

## ローテーション方針

- 期限ベースまたはインシデント発生時にローテーションする
- ローテーション後は旧キーを速やかに無効化する
- 監査ログで変更履歴を追跡できる状態を維持する

## インシデント時の最小手順

1. 漏えいしたキーを即時無効化する
2. 新しいキーを発行して各環境へ再配布する
3. 影響範囲を調査し、必要な通知を行う
4. 再発防止策を文書化する
